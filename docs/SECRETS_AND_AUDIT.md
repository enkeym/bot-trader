# Секреты, аудит, идемпотентность

## Хранение секретов

- Используйте только [`.env`](../.env.example) на сервере; файл **не коммитится**.
- Ключи Binance: минимальные права, без вывода на неизвестные адреса; при возможности IP whitelist.
- `TELEGRAM_BOT_TOKEN` храните как любой секрет (Vault, Docker secrets, CI variables).

## Аудит

- [`AuditService`](../src/audit/audit.service.ts) пишет в таблицу `AuditLog` и дублирует в stdout.
- Критичные действия (симуляции, отказ риска) логируются через `audit.log()`.
- В логи **не** попадают полные банковские реквизиты (при расширении не логируйте PAN/полные номера карт).

## Идемпотентность ордеров

- [`OrderIntentService`](../src/order/order-intent.service.ts): поле `idempotencyKey` уникально в БД; повтор с тем же ключом возвращает существующую запись (защита от дублей при ретраях).
- Симуляция использует детерминированный ключ на минутный bucket + округлённый спред.
