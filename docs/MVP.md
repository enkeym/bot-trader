# MVP: объём и режимы

## Зафиксированный scope первой версии

1. **Мониторинг P2P** — опрос публичного стакана USDT/RUB (Binance), расчёт потенциального спреда (лучший SELL vs лучший BUY с учётом комиссий из конфига).
2. **Симуляция** — при `DRY_RUN=true` виртуальные «сделки» в БД без Spot API. При `DRY_RUN=false` и ключах — **Spot MARKET** на `api.binance.com`.
3. **Telegram** — команды `/start`, `/spread`, `/status`; уведомления при превышении порога спреда (если включено).
4. **Риск-лимиты** — дневной лимит виртуального убытка, минимальный спред %, макс. объём (конфиг + `RiskService`).
5. **TON** — опциональная проверка доступа по TON Connect / подписке (минимальная заготовка).

## Режимы исполнения (`EXECUTION_MODE`)

| Режим | Поведение |
|-------|-----------|
| `AUTO_EXCHANGE_ONLY` | Чтение P2P, логика, при `DRY_RUN=true` — только запись в БД; при `DRY_RUN=false` и ключах — **Spot MARKET** (см. README). P2P по API не исполняется. |
| `REQUIRE_HUMAN_BANK_CONFIRM` | После сигнала стратегии бот **не** считает сделку завершённой, пока оператор не подтвердил банковский шаг вручную (кнопка в Telegram или внешний флаг в БД). |

Переменная окружения: `EXECUTION_MODE` = `AUTO_EXCHANGE_ONLY` | `REQUIRE_HUMAN_BANK_CONFIRM`.

## Реальные микро-сделки (Spot)

Spot: `DRY_RUN=false` + `BINANCE_API_KEY` / `BINANCE_API_SECRET` + `BINANCE_SPOT_*` (символ, лимит quote и т.д.). Рекомендуется начинать с `DRY_RUN=true` для проверки логики.
