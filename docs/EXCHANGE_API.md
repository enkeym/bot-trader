# Выбор P2P API (валидация для RUB / СНГ)

## Рекомендуемая основная площадка: **Binance P2P (C2C)**

Обоснование для MVP мониторинга и симуляции:

| Критерий | Binance P2P |
|----------|-------------|
| Ликвидность USDT/RUB | Высокая среди глобальных площадок |
| Публичный поиск объявлений | Да, без API-ключа (`/bapi/c2c/v2/friendly/c2c/adv/search`) |
| Подписанные операции | SAPI `sapi/v1/c2c/...` — требуют API key + права; часть эндпоинтов только для **merchant** |

### Публичный рынок (без ключа)

- **POST** `https://p2p.binance.com/bapi/c2c/v2/friendly/c2c/adv/search`
- Тело: `asset` (напр. `USDT`), `fiat` (`RUB`), `tradeType` (`BUY` | `SELL`), `page`, `rows`
- Используется в адаптере `src/p2p/providers/binance-c2c.provider.ts` для лучших цен и оценки спреда

### Подписанные вызовы (HMAC SHA256)

- База: `https://api.binance.com` (Spot API) и отдельные SAPI пути для C2C
- Требования: ключ с минимальными правами; **не включать вывод** без необходимости
- Лимиты: см. [Binance API](https://binance-docs.github.io/apidocs/spot/en/) — weight и IP limits

### Альтернатива для второй фазы

- **Bybit** — P2P REST в документации Bybit; структура отличается → отдельная реализация `P2PProvider`
- **OKX** — аналогично, свой REST

Интерфейс `P2PProvider` в коде позволяет подключить вторую биржу без смены стратегии.

## Ограничения автоматизации

- Управление объявлениями и сделками по API зависит от статуса аккаунта (в т.ч. merchant) и актуальных правил биржи.
- Банковский перевод RUB (СБП/карта) не автоматизируется биржей полностью; в приложении заложены режимы `AUTO_EXCHANGE_ONLY` и `REQUIRE_HUMAN_BANK_CONFIRM` (см. [MVP.md](./MVP.md)).
